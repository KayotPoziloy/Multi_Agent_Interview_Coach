# Multi_Agent_Interview_Coach

# Отчет по проекту: Multi-Agent Interview Coach

## Введение

**Цель работы** — разработать мультиагентную систему, проводящую техническое интервью и имитирующую работу реального рекрутера за счет:

- ролевой специализации агентов,
- скрытой рефлексии (внутреннего анализа),
- адаптации сложности вопросов,
- устойчивости к галлюцинациям и off-topic ответам,
- генерации полезного структурированного фидбэка.

Проект выполнен в рамках задания *Multi-Agent Interview Coach* с использованием Python и LLM (Google Gemini через LangChain).

---

## Архитектура решения

Система реализована как **мультиагентный workflow**, состоящий из трех специализированных агентов:

### Interviewer Agent

**Ответственность:**

- ведение диалога с кандидатом,
- постановка технических вопросов,
- адаптация сложности вопросов,
- вежливое управление off-topic ситуациями,
- ответы на встречные вопросы кандидата.

Interviewer **не занимается** проверкой фактов и **не принимает** решение о найме.

---

### Observer Agent

**Ответственность:**

- скрытый анализ ответов кандидата,
- факт-чек (распознавание ложных утверждений и галлюцинаций),
- оценка уверенности и глубины знаний,
- выдача инструкций Interviewer Agent.

Observer полностью **скрыт от пользователя**. Его выводы логируются в `internal_thoughts`.

---

### Manager Agent

**Ответственность:**

- анализ полного лога интервью,
- финальная оценка hard и soft skills,
- определение грейда и рекомендации по найму,
- формирование персонального roadmap кандидата.

Manager вызывается **только на этапе завершения интервью**.

---

## Соответствие требованиям Agentic Workflow

### Role Specialization

Каждый агент имеет строго ограниченную роль:

- Interviewer — диалог и вопросы,
- Observer — анализ и контроль качества ответов,
- Manager — итоговое решение.

Таким образом, система не является одним большим промптом.

---

### Hidden Reflection

Перед каждым ответом Interviewer Agent происходит скрытый этап:

1. Observer анализирует ответ кандидата.
2. Формирует структурированную инструкцию.
3. Interviewer генерирует следующий ход с учетом этой инструкции.

**Пример (в логах):**

```
[Observer]: Обнаружена галлюцинация. Утверждение про Python 4.0 ложно.
[Interviewer Decision]: Нужно мягко указать на ошибку и вернуть к теме.
```

---

### Context Awareness

Контекст последних N ходов передается агентам при каждом запросе.

**Гарантии:**

* агент не задает повторные вопросы,
* учитывает информацию из приветствия и прошлых ответов,
* корректно реагирует на встречные вопросы.

---

### Adaptability

Сложность вопросов изменяется динамически:

* уверенные и корректные ответы → усложнение,
* неуверенность или ошибки → упрощение или уточняющие вопросы.

Решение об адаптации принимает Observer Agent.

---

### Robustness

Система устойчива к:

* **галлюцинациям пользователя** (например, мифы про Python 4.0),
* **off-topic ответам**,
* **role reversal** (кандидат задает вопрос интервьюеру).

Interviewer не соглашается с бредом и не игнорирует вопросы кандидата.

---

## Логирование и формат хранения данных

Система сохраняет интервью в JSON-файл следующей структуры:

```json
{
  "participant_name": "Алекс",
  "position": "Backend Developer",
  "target_grade": "Junior",
  "turns": [
    {
      "turn_id": 1,
      "agent_visible_message": "Привет! Расскажи о себе",
      "user_message": "Я знаю Python и SQL",
      "internal_thoughts": "[Observer]: Базовый уровень"
    }
  ],
  "final_feedback": { ... }
}
```

**Сюда в финальной версии реальный лог прогона системы.**

---

## Финальный фидбэк

После команды завершения интервью система генерирует строго структурированный JSON-отчет, включающий:

* итоговый грейд,
* рекомендацию по найму,
* confidence score,
* confirmed skills и knowledge gaps (с правильными ответами),
* оценку soft skills,
* персональный roadmap.

**Сюда JSON-фидбэк, полученный после прогона сценария.**

## Выводы

В рамках проекта была реализована полноценная мультиагентная система, удовлетворяющая требованиям Agentic Workflow.

Решение демонстрирует:

* осмысленное поведение,
* разделение ответственности,
* интерпретируемые логи,
* практическую пользу как для кандидата, так и для рекрутера.
